RMT peripheral (Remote Control Transceiver), DMA, or a library like FastLED or Adafruit NeoPixel.

RMT peripheral is a module from ESP32 designed for precised pulse generation and reception.

TX mode: transmit
RX mode: receive

Why RMT (TX mode) ?
- I need timing extremly precise
- Pulse timing is generated by hardware (not CPU)

doc:
https://docs.espressif.com/projects/esp-idf/en/stable/esp32/api-reference/peripherals/rmt.html


## Initialisation

rmt_channel_handle_t rx_chan = NULL;
rmt_rx_channel_config_t rx_chan_config = {
    .clk_src = RMT_CLK_SRC_DEFAULT,   // select source clock
    .resolution_hz = 1 * 1000 * 1000, // 1 MHz tick resolution, i.e., 1 tick = 1 µs
    .mem_block_symbols = 64,          // memory block size, 64 * 4 = 256 Bytes
    .gpio_num = 2,                    // GPIO number
    .flags.invert_in = false,         // do not invert input signal
    .flags.with_dma = false,          // do not need DMA backend
};
ESP_ERROR_CHECK(rmt_new_rx_channel(&rx_chan_config, &rx_chan));
